## Design Context

### Users
Solo developer/entrepreneur with packed schedule mixing Islamic prayers, company work (GitHub merges), side projects (Wingmic), and personal logistics (moving, family). Uses the tool daily to plan tomorrow's schedule by speaking to an AI agent. Context: end-of-day planning session, often tired, needs clarity fast.

### Brand Personality
Focused, spiritual, urgent

### Emotional Goals
Premium creative fusion — calm command center meets urgent mission control meets minimal zen. Should feel like a high-end cockpit designed by a creative studio: controlled pressure with moments of calm, wrapped in a polished, slightly funky aesthetic. Not corporate, not bland, not stressful.

### Aesthetic Direction
- **Theme**: Dark-first glassmorphism. Deep navy/midnight backgrounds with frosted glass panels.
- **Vibe**: Premium + creative funky. Clean lines but with personality — gradient accents, subtle glow effects, confident typography.
- **Color palette**: Navy base (#070b17, #10182f), cool blue accent (#7aa2ff), emerald green for success (#32d6a0), amber for warnings (#ffd166), coral for danger (#ff6b81). Semi-transparent whites for glass panels.
- **Typography**: Inter (system sans-serif fallback). Tabular nums for timers/clocks. Monospace for metrics/code data.
- **Radius**: 12-18px for cards, 999px for pills/badges.
- **Effects**: backdrop-filter blur(8px), deep box shadows (0 12px 34px), subtle border glow on interactive elements.
- **Motion**: Smooth transitions (150-300ms ease). Progress bar fills. Toast slide-ups. No excessive animation.
- **Anti-references**: None specified. Avoid corporate/enterprise aesthetics (Jira, Salesforce). Avoid flat/bland (plain Bootstrap).

### Design Principles

1. **Clarity over decoration** — Every element earns its space. If it doesn't help the user act, it goes.
2. **Urgency without anxiety** — Time pressure is visible (progress bars, countdowns) but presented as empowering, not stressful.
3. **Spiritual anchoring** — Prayer blocks are first-class citizens, visually distinct, never treated as optional.
4. **Ship over polish** — Prefer working features over perfect pixels. Good enough today beats perfect next week.
5. **One-glance comprehension** — Dashboard state (what's now, what's next, how far through the day) readable in under 3 seconds.

### Component Patterns

- **Cards**: `.card` — glass panel with blur, 18px radius, 1px border, deep shadow
- **Pills/Badges**: `.pill` — 999px radius, small font, border + tinted background. Variants: `.good`, `.warn`
- **Buttons**: Default glass + border. `.primary` = blue gradient. `.good` = green gradient. Hover lifts -1px.
- **Inputs**: Dark inset background, 10px radius, focus ring with --ring color
- **Timeline items**: Grid layout (time | content | action), state classes: `.current`, `.done`, `.missed`
- **Toast**: Fixed bottom-right, slide-up reveal, auto-dismiss 3.4s
- **Progress bars**: 10px height, rounded, gradient fill (blue to green)
- **Layout**: 12-column CSS grid, max 1100px, 16px gap. Responsive collapse at 930px.
